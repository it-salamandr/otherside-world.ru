БК-2 { } Схемокодер (прототип)

[вводная]

Это прототип версии программы «Схемокодер» на примере базового составления схемы html+js, созданного по мотивам книги Запретная зона Схемокодеров .

В её основе лежит возможность собирать визуальные схемы (блоков, контейнеров и связей) и на их основе формировать HTML-страницу: элементы добавляются, удаляются, редактируются; контейнеры получают статус модульных, поддерживаются команды «перейти к элементу N», «подключить модульный контейнер N».

Проект полностью в тестовом режиме: выбор возможностей ограничен, например, пока можно только задавать стили и текстовые атрибуты.

Цель программы — дать пользователю возможность «программировать» через визуальные схемы, упорядочивать структуру элементов, видеть зависимости и видеть, как по итогу генерируется код в виде массива.

Выбирайте сверху тип схемы (кнопки: Контейнер, Атрибут, Рефлекс, Функционал) и вводите ниже для неё данные (пример на ютубе):
Собираться схема будет здесь.

Тестовый запуск пока включает в себя:
1. последовательный авто-проход по элементам (игнор контейнеров на 1-м уровне).
2. для рефлекса понимает команду "продолжить с другого элемента": перейти на {id: N}
3. контейнеры в корне программы "игнорируются авто-проходом" и им присваивается статус модульный: TAG_NAME модульный
4. для рефлекса понимает команду "заинклудить модульный контейнер": подключить {id: N}
p.s. значение N заменять на число, например: подключить {id: 23}
p.s. максимум вложенных инклудов (для тестового запуска): 2.
p.s. максимум зациклиенных переходов (для сформированной страницы): 10.
p.s. фукционал не выполняется
p.s. в атрибуты можно пока добавлять только стили и innerText
p.s. в планах 2-я версия триггеров, 3-я версия связей и 4-я итоговая привилегий.
p.s. в атрибуты можно добавить рефлексы (автозапуск их не запустит, срабатает при контакте с пользователем)
